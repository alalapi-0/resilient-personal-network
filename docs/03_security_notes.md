# 03 安全说明（Security Notes）

本文件用于约束敏感信息管理方式。请在每次提交前对照检查。

## 必须遵守的原则

1. 不要把服务器 IP、真实域名、UUID、私钥直接提交到 GitHub。  
2. 不要把真实客户端链接公开。  
3. `.env` 文件不应提交。  
4. `backups` 目录默认不提交真实备份。  
5. `configs/server/*.json` 默认不提交真实服务端配置。  
6. `configs/client/*.json` 默认不提交真实客户端配置。  
7. `nodes/*/*.env` 默认不提交。  
8. 分享代码前要检查敏感信息。  
9. 不要在截图中暴露服务器地址和密钥。  
10. 不要把同一组密钥用于多个节点。  

## 提交前快速自检建议
1. 先运行 `git status` 查看将提交的文件。
2. 再运行 `git diff --staged` 检查是否出现真实域名、IP、密钥。
3. 若发现敏感内容，立即移除并改用占位符或 `.env` 引用。

## 本项目在 Round 0 的安全边界
- 仅允许模板与文档入库；
- 禁止入库任何可直接连接真实节点的配置；
- 所有示例值都应是教学占位符。
