# 02 目标架构说明（Architecture）

## 未来目标架构（示意）

```text
手机 / Mac / 平板
        |
        v
客户端配置管理
        |
        v
主节点 jp-tokyo-01
        |
        +---- 备用节点 sg-01
        |
        +---- 备用节点 us-la-01
```

> 说明：上图是逻辑结构示意，不代表当前 Round 0 已完成真实部署。

## 1. 本地仓库负责什么
本地仓库（也就是本项目）负责：
- 管理目录结构、脚本和文档；
- 保存配置模板（不是敏感真实配置）；
- 记录节点规划与运维流程；
- 支持后续扩展为 CLI 工具。

## 2. VPS 服务器负责什么
VPS 节点负责：
- 运行服务端程序；
- 对外提供可访问入口；
- 承担数据转发。

在未来阶段，主节点承担主要流量，备用节点用于故障切换。

## 3. 客户端负责什么
客户端（手机/电脑/平板）负责：
- 加载配置；
- 与服务端建立连接；
- 在链路不稳定时按策略切换节点（后续实现）。

## 4. 文档负责什么
文档负责“知识沉淀与流程标准化”：
- 解释为什么这样设计；
- 约定统一术语与命名；
- 记录排障步骤与变更历史；
- 帮助新手快速接手。

## 5. 备份负责什么
备份机制负责：
- 保存关键模板、脚本、节点元信息；
- 在误改、误删、升级失败时提供恢复依据；
- 形成可追溯的版本管理与应急恢复流程。
